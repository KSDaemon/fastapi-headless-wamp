- Typecheck: `uv run pyright src/` (strict mode, Python 3.12 target)
- Tests: `uv run pytest` with pytest-asyncio in auto mode
- README.md is required by hatchling build (even if minimal)
- pyright config is in pyproject.toml under `[tool.pyright]`
- In pyright strict mode, use typed factory functions for dataclass default_factory (e.g., `def _dict_factory() -> dict[str, Any]: return {}`) instead of bare `dict`/`list`
- Message dataclasses use deferred imports inside `to_list()` methods to avoid circular imports between types.py and protocol.py
- `validate_message()` takes `Any` (not `list[Any]`) so the isinstance check is meaningful at runtime
- WampSession wraps starlette WebSocket + Serializer; uses text mode for JSON, binary mode for binary serializers
- Tests use a `MockWebSocket` class with async queues for simulating send/receive without a real server

- `negotiate_subprotocol()` is a standalone function in session.py that matches client subprotocols against the serializer registry
- WampHub.handle_websocket() is the complete async handler: negotiate subprotocol → accept → handshake → message loop → cleanup
- Session handshake: `session.perform_handshake(realm, existing_ids)` handles HELLO → WELCOME, ABORT on invalid first message, realm mismatch
- GOODBYE handling: `session.handle_goodbye(msg)` sends GOODBYE reply with `wamp.error.goodbye_and_out`
- Hub tracks sessions in `_sessions: dict[int, WampSession]`; session added after WELCOME, removed in finally block
- `generate_session_id(existing_ids)` retries to ensure uniqueness among active sessions
- Lifecycle callbacks: `@hub.on_session_open` / `@hub.on_session_close` are decorators on WampHub that register async callbacks; they fire in order and catch exceptions to avoid crashing the hub
